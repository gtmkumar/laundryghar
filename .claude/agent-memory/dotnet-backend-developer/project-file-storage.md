IFileStorageProvider is in laundryghar.ServiceDefaults/Storage/ and registered via AddServiceDefaults() — no per-service wiring. Default is local (writes under Storage:Local:RootPath, default /tmp/laundryghar-uploads). S3 and azure-blob are documented throw-seams in FileStorageProviderFactory.

Key scheme: `{brandId:N}/{area}/{uuid:N}.{ext}` — always server-generated by FileStorageKeyGenerator. Area must match `[a-z0-9][a-z0-9_/\-]{0,63}` (first char must be letter/digit to block leading-slash paths). MIME→ext map in KeyGenerator.

.NET 10 multipart endpoints with IFormFile require `.DisableAntiforgery()` on the route builder. Body size cap uses `.WithMetadata(new RequestSizeLimitAttribute(...))` — `WithRequestSizeLimit()` does NOT exist as an extension on RouteHandlerBuilder.

Inspection photos endpoint: POST /api/v1/admin/garment-inspections/{id}/photos (permission:garment.inspect). Streams back by photo id at GET .../garment-inspections/photos/{photoId} (no raw key exposed).

Rider PoD: POST /api/v1/rider/tasks/{id}/proof-photo (RiderOnly). Admin streams at GET /api/v1/admin/rider-tasks/{id}/proof-photo (permission:rider.read). Stores in delivery_assignments.proof_photo_s3_key + proof_photo_taken_at (column added by patch proof_photo_taken_at.sql, applied 2026-06-10).

Worker.Tests has pre-existing build errors (AnonymizeCustomer not found) — these pre-dated this task and are in a different lane.

**Why:** Photo evidence is a compliance/operations requirement for the rider PoD and warehouse QC flows. The local provider is intentionally not production-suitable — its purpose is dev-loop only; S3/azure-blob seams make production migration a one-file change.

**How to apply:** When adding new upload endpoints always use the same pattern: generate key server-side, store key in DB, stream via an ID-based endpoint (never expose raw keys in the API response).
